# QuranAI Browser Extension

This directory contains the source code for the QuranAI browser extension, built using Svelte.

## Architecture

- **`index.html`**: The entry point for the extension UI (sidebar/sidepanel). It loads the bundled JavaScript and CSS.
- **`index.js`**: The entry point for the extension script, responsible for mounting the Svelte `App` component.
- **`content.js`**: Content script injected into `alislam.org/quran/app` to provide verse-level interactions. It detects Quran verses and injects two Material Icons (➕ and 💡) for context management and quick explanation.
- **`manifest.json`**: Extension manifest (v3) configured for cross-browser compatibility.
- **`background.js`**: Background script/service worker for panel behavior and extension API management.
- **`bundle.js`**: Bundled JavaScript generated by Rollup, including the Svelte runtime and application code.
- **`bundle.css`**: Extracted CSS from Svelte components, generated by Rollup.

## Svelte Migration

The UI elements have been migrated to Svelte for better state management and component reusability. The core UI logic resides in `src/frontend/ui/App.svelte` and is shared with the standalone web application.

## Verse Interaction Features

- **Add to Context (➕)**: Click the "plus" icon next to any verse on the page to add it to your current chat context. The verse reference will appear as a badge in the sidebar, and subsequent questions will be answered with that verse's context in mind.
- **Explain Verse (💡)**: Click the "lightbulb" icon to instantly trigger a query for the agent to explain that specific verse.

## Sidebar Visibility & State Management

To provide a seamless experience where in-page icons are only visible when the extension's sidebar is open, the extension implements a robust state management system:

1. **Heartbeat Port**: The `App.svelte` component opens a persistent `runtime.connect` port (named `quranai-sidepanel`) to the background script on mount. When the sidebar is closed or unmounted, the port automatically disconnects.
2. **State Tracking**: `background.js` listens for this connection to track the sidebar's `isOpen` state globally.
3. **Broadcasting**: Whenever the state changes, `background.js` sends a `sidepanel_state_changed` message to all active tabs.
4. **Content Script Reactivity**: `content.js` maintains a local `isSidePanelOpen` flag. 
    - **When Closed**: It removes all injected icons and resets its `data-quranai-injected` tracking attributes.
    - **When Opened**: It triggers a fresh scan of the page to inject icons.
    - **Mutation Monitoring**: It uses a `MutationObserver` to ensure icons are injected into newly loaded content while the sidebar is open.

## Cross-Browser Compatibility (Manifest V3)

The `manifest.json` is configured to support both Chrome and Firefox as a persistent sidebar:

### Chrome-Specific Settings
- **`permissions: ["sidePanel"]`**: Required to use the `chrome.sidePanel` API.
- **`side_panel`**: Defines the default path for the Chrome Side Panel.
- **`background.service_worker`**: Chrome MV3 requires background logic to run in a Service Worker.

### Firefox-Specific Settings
- **`sidebar_action`**: Defines the UI for the Firefox Sidebar (Firefox doesn't use the `side_panel` key).
- **`background.scripts`**: Firefox MV3 uses a background script array.
- **`browser_specific_settings.gecko`**: Required for Firefox MV3 extensions to provide a unique ID.
- **Programmatic Toggle**: In Firefox, the extension icon click is handled in `background.js` to toggle the sidebar. This must be done **synchronously** in the click handler to satisfy "user input handler" security requirements.

## Build

The extension uses Rollup for bundling. To rebuild:

```bash
npm run build
```

## Local API Integration

The extension connects to `http://localhost:8000/`. Ensure the QuranAI API server is running locally.
